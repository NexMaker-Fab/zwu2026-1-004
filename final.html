FINAL PROJECT

基于ESP32-S3的老年人智能健康手表

一、项目概述

1.1 项目名称

基于ESP32-S3的老年人智能健康手表

1.2 项目定位

本作品是一款专为独居、空巢老人设计的智能健康手表，以ESP32-S3为主控核心，集成心率监测、异常报警、震动提醒、WiFi网页数据查看等功能，旨在解决老年人突发健康风险无人知晓、子女无法远程监护的核心痛点。

1.3 设计理念

  • 适老性优先：遵循适老化设计原则，外形简单易懂，采用大按键、圆角设计、高对比度显示
  • 轻量化穿戴：参考成熟老年手表产品设计经验，表盘厚度控制在13-15mm，减轻长时间佩戴负担
  • 操作零门槛：触屏交互为主，一键SOS为辅，无需复杂设置即可使用
  • 远程守护：子女通过手机浏览器即可实时查看老人心率数据，实现远程监护
图1：老年人智能健康手表整体外观

二、项目背景与意义

2.1 社会背景

我国老龄化进程加快。据《2023年度国家老龄事业发展公报》，2023年末我国60周岁及以上老年人口达2.97亿，占总人口的21.1%。与此同时，独居老人数量持续增加，因突发疾病或意外无人及时发现导致的悲剧时有发生。

2.2 市场痛点

当前智能手表市场主流产品功能设计多倾向于年轻消费群体，对高龄用户而言，时尚外观与复杂操作界面往往成为使用障碍，诸多冗余功能反而降低了产品的使用价值。专门面向老年人的健康监测手表存在显著市场空白。

2.3 项目意义

本作品以低成本的硬件方案（总预算约250元），实现心率实时监测、异常报警、远程数据查看等核心功能，为老年人提供一种便携、易用、可靠的健康监护方案，同时为子女提供远程关怀的窗口。

四、AI功能方向

方向1：AI语音助手（最适合老年人）

核心场景：老人直接说话，手表听懂并回应。

老人说 手表回应
"今天天气怎么样？" 播报天气（通过WiFi获取）
"现在几点了？" 语音报时
"提醒我吃药" "好的，几点吃？"→设定定时提醒
"我儿子电话多少？" 显示/播报紧急联系人号码
"我有点不舒服" 自动触发报警+震动，并语音安慰

用户价值：老人不需要学习任何操作，直接说话就能用。完美解决"操作零门槛"的设计理念。

方向2：AI摔倒检测（独居老人最需要）

核心场景：老人摔倒后无法起身，手表自动识别并报警。

实现方式：

  • 利用ESP32-S3内置的加速度传感器（MPU6050，外接）
  • 检测到"突然跌落+静止不动"模式 → 自动触发蜂鸣器+震动+WiFi通知

用户价值：即使老人说不出话，手表也能主动求救。

方向3：AI情绪识别与安慰

核心场景：老人长时间不说话或心率异常波动，手表主动关怀。

实现方式：

  • 心率变异性（HRV）分析 → 判断情绪状态
  • 长时间无交互（2小时无按键+无人靠近）→ 主动播放语音安慰或亮灯提示

用户价值：缓解独居老人的孤独感，实现"情感陪伴"理念。

六、技术实现

6.1 心率监测实现

  1. MAX30102传感器通过I2C接口与ESP32-S3通信
  2. 采集原始光电信号，滤波处理后计算心率值
  3. 实时显示在OLED屏幕上
  4. 数据存储至内存，供远程查看

6.2 异常报警机制

  1. 设定心率阈值：正常范围60-100次/分钟
  2. 超出阈值时，同时触发蜂鸣器和震动马达
  3. 通过WiFi发送报警信息到云端
  4. 子女端网页实时接收报警通知

6.3 WiFi远程查看

  1. ESP32-S3开启AP模式，创建WiFi热点
  2. 搭建简易Web服务器，托管HTML页面
  3. 手机连接热点后，浏览器访问指定IP
  4. 网页实时显示心率曲线和历史数据
