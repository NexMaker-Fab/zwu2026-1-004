开发历程

阶段 1：需求分析与规划

时间：项目启动初期

我们首先明确了网站的目标用户和核心功能需求。作为 Fab Academy 的作业提交平台，需要包含首页、日常作业、最终项目和团队介绍等模块。团队进行了多次讨论，确定了网站的整体架构和功能清单。

  • 确定网站目标和受众
  • 列出核心功能需求
  • 制定项目开发计划
  • 分配团队成员职责

阶段 2：初版设计与开发

时间：第一周

团队开始搭建网站的基础框架，创建了第一个版本的 HTML 结构和 CSS 样式。这个阶段我们重点关注功能的完整性，确保所有页面都能正常访问和导航。

  • 创建基础 HTML 结构
  • 编写初始 CSS 样式
  • 实现页面导航系统
  • 添加基本的内容布局

阶段 3：高端设计重构

时间：第二周

为了提升用户体验和视觉吸引力，我们决定对网站进行全面的设计升级。引入了玻璃态设计（Glassmorphism）、渐变配色方案和流畅的动画效果，让网站更具现代感和专业感。

  • 采用深色主题背景
  • 实现玻璃态卡片效果
  • 添加渐变色和动画
  • 优化响应式布局
  • 增强交互体验

阶段 4：团队页面开发

时间：第三周

我们创建了专门的团队介绍页面，展示 Aether Studio 的 7 位成员。每位成员都有独特的渐变色头像和个人简介，体现了团队的多样性和创造力。

  • 设计团队成员卡片
  • 添加个人简介和社交链接
  • 创建团队理念介绍
  • 展示团队优势和特色

阶段 5：GitHub 部署配置

时间：第四周

为了让网站能够在线访问，我们配置了 GitHub Actions 自动化部署流程。每次代码推送到仓库后，都会自动构建并部署到 GitHub Pages，实现了持续集成和持续部署（CI/CD）。

  • 创建 GitHub Actions 工作流
  • 配置自动化部署脚本
  • 设置 Pages 发布源
  • 测试部署流程
  • 编写部署文档

阶段 6：问题修复与优化

时间：持续进行

在开发过程中，我们发现并修复了 CSS 文件引用路径的问题，确保所有页面都能正确加载样式。同时不断优化代码结构，提升网站性能和可维护性。

  • 统一 CSS 文件引用路径
  • 修复样式加载问题
  • 优化代码结构
  • 完善文档说明

设计亮点

✨ 玻璃态效果

使用 backdrop-filter 实现毛玻璃效果，配合半透明背景和模糊处理，营造出层次感和现代感。

🌈 渐变配色

精心调配的紫色、粉色、蓝色渐变方案，每个元素都有独特的色彩组合，视觉冲击力强。

🎭 流畅动画

卡片悬停、按钮点击、页面滚动等交互都有平滑的过渡动画，提升用户体验。

📱 响应式设计

使用 CSS Grid 和 Flexbox，确保网站在手机、平板、桌面等各种设备上都能完美显示。

遇到的挑战与解决方案

🔧 挑战 1：CSS 文件引用问题

问题：部分 HTML 文件引用了错误的 CSS 文件名（style-premium.css），导致样式无法加载。

解决：统一所有文件的 CSS 引用为 style.css，确保路径正确。

🎨 挑战 2：设计风格统一

问题：如何在多个页面中保持设计的一致性。

解决：使用 CSS 变量定义颜色、间距等，创建可复用的组件样式。

📱 挑战 3：响应式适配

问题：确保网站在不同屏幕尺寸下都能良好显示。

解决：使用媒体查询和弹性布局，针对移动端优化导航和时间线显示。

🚀 挑战 4：自动化部署

问题：配置 GitHub Actions 实现自动部署。

解决：研究 GitHub Actions 文档，创建 deploy.yml 工作流文件，成功实现推送即部署。
